Clases con significado Moléculas

Molecules

Recordemos las moléculas en Atomic Design son conjuntos de elementos simples (átomos) que, al combinarse, forman un componente con más sentido y significado, aunque no son completamente independientes.

Detalles a tener en cuenta en una molécula

Átomos ligados a una molécula

EJEMPLO 1

En algunos casos, ciertos elementos de una molécula no se usarán en ningún otro lugar del proyecto. Por ejemplo, en una molécula de "video-thumbnail" (que podría ser una vista previa de video con un botón de reproducción), el botón de play es parte integral de esta molécula y no necesita ser definido como un átomo separado.

 <div class="m-video-thumbnail">
    <img
      alt="Vídeo introducción"
      src="img/bg/cursos.png"
      class="m-video-thumbnail__img"
    />
<!-- Aquí tenemos nuestro boton de reproducción -->
    <button
      aria-label="Reproducir Vídeo introducción"
      class="m-video-thumbnail__play m-video-thumbnail__play--large"
    ></button>
</div> 
 /*File video-thumbnail.css*/
.m-video-thumbnail {
  position: relative;
}
 
.m-video-thumbnail__img {
  display: block;
  margin-bottom: 1.6rem;
  border-radius: 0.5rem;
}
 
/* No consideramos el botón de "play" como un átomo independiente 
porque está estrechamente vinculado al diseño específico de la portada 
del video. Un átomo, por definición, debería ser reutilizable en diferentes 
contextos, pero este botón en particular no tendría utilidad fuera de esta 
situación específica. Por eso, no lo tratamos como un átomo separado. 
En lugar de usar el prefijo a-, utilizamos m-, indicando 
que este botón forma parte de la molécula m-video-thumbnail. 
De esta manera, señalamos que el botón está 
vinculado específicamente a esta molécula y no es un elemento reutilizable 
en otros contextos.*/
.m-video-thumbnail__play {
  position: absolute;
  top: 50%;
  left: 50%;
  display: block;
  width: 5em;
  height: 5em;
  border: none;
  background: var(--background-lighter-color);
  border-radius: 50%;
  cursor: pointer;
  transform: translate(-50%, -50%);
}
 
.m-video-thumbnail__play::before {
  position: absolute;
  top: 50%;
  left: 50%;
  width: 0;
  height: 0;
  border-width: 6px 0 6px 12px;
  border-style: solid;
  border-color: transparent transparent transparent var(--background-dark-color);
  margin-top: -5px;
  margin-left: -4px;
  content: "";
}
 
.m-video-thumbnail__play--large {
  width: 7.4em;
  height: 7.4em;
}
 
.m-video-thumbnail__play:hover {
  box-shadow: 0 23px 40px rgba(0, 0, 0, 0.2);
}

EJEMPLO 2

 <form class="m-form">
   <label for="name" class="m-form__label">Nombre:</label>
   <input type="text" id="name" class="m-form__input-text">
   <button class="m-form__btn m-form__btn--primary">Enviar</button>
</form>
 /*File form.css*/
.m-form {
   max-width: 400px;
   margin: 0 auto;
   padding: 20px;
   border: 1px solid #ccc;
   border-radius: 8px;
   background-color: #f9f9f9;
}

.m-form__label {
   display: block;
   margin-bottom: 8px;
   font-weight: bold;
}

.m-form__input-text {
   padding: 8px;
   border: 1px solid #ccc;
   border-radius: 4px;
   width: 100%;
   box-sizing: border-box;
   margin-bottom: 16px;
}

.m-form__input-text:focus {
   border-color: #007BFF;
   outline: none;
}

.m-form__btn {
   padding: 10px 20px;
   background-color: #007BFF;
   color: white;
   border: none;
   border-radius: 4px;
   cursor: pointer;
}

.m-form__btn--primary {
   background-color: #007BFF;
}

.m-form__btn:hover {
   background-color: #0056b3;
}

Explicación de la Estructura:

  • m-form: La molécula completa del formulario, con estilos que afectan a la estructura global.

  • m-form__label, m-form__input-text, m-form__btn: Estos son elementos que pertenecen exclusivamente a la molécula m-form. El uso del prefijo m-form__ señala que estos estilos no están destinados a ser reutilizados fuera de esta molécula específica.

Modularidad vs. Especificidad:

  • Modularidad (Átomos Reutilizables): Si los componentes como label, input, o button fueran a reutilizarse en otros formularios o partes de la UI, entonces deberías mantener los prefijos a- (por ejemplo, a-label, a-input-text, a-btn), asegurando que sean estilos genéricos y reutilizables.

  • Especificidad (Elementos Ligados a una Molécula): Si esos elementos están diseñados solo para esta molécula, asignarles un prefijo m- (como en m-form__label) es una manera eficaz de indicar que pertenecen exclusivamente a esta estructura.

Selectores más específicos

A este nivel del desarrollo, ya estamos cerca de la cima del triángulo invertido de ITCSS. Esto significa que es un buen momento para empezar a utilizar selectores más específicos, siempre respetando la regla de ir de lo menos específico a lo más específico.

Veamos la siguiente situación: 

 <div class="a-box m-cta-box">
 <h3 class="m-cta-box__title">
   Suscripción mensual <strong>29€</strong>
 </h3>
 <span class="m-cta-box__note">Sin permanencia mínima</span>
 <ul class="o-ui-list m-cta-box__content">
   <li>💻 Acceso a todos los cursos ya publicados</li>
   <li>
     👌 Contenido de calidad impartido por profesionales con amplia
     experiencia
   </li>
   <li>🚀 Nuevo contenido cada semana</li>
   <li>🙌 Acceso a la comunidad</li>
   <li>🔖 Certificados al completar cursos</li>
 </ul>
 <a
   class="a-btn a-btn--primary m-cta-box__cta"
   href="#">
   <span class="a-btn__text">👉 SUSCRÍBETE AHORA 👈</span>
 </a>
</div>

Lo que pasa es el encontrar la manera de lograr estilar 

/*File cta-box.css*/
.m-cta-box {
 position: relative;
 padding-bottom: 4rem;
}

.m-cta-box__title {
 padding-bottom: 1em;
 border-bottom: 2px solid rgba(var(--border-inverted-color), 0.3);
 margin: 0;
 margin: 0.5em 0 1em;
}

/*Como ya estamos en la parte final de ITCSS ya podemos utilizar selectores 
más específicos, por ejemplo en este caso hemos utilizando un 
selector descendiente */
.m-cta-box__title strong {
 display: block;
}

.m-cta-box__note {
 position: absolute;
 top: 1.5rem;
 right: 1.5rem;
 font-size: var(--fs-pill);
 text-transform: uppercase;
}

/*En este caso pudimos haber estilado tambien con 
un selector descendiente tomando la molécula y descendiendo 
hasta este botón, pero hemos optado por crear una 
clase nueva como elemento de esa molécula que seria 
.m-cta-box__cta*, hay que saber que no hay solución que siempre 
sea la misma, la solución es muy subjetiva solo hay que tomar una 
y a seguir para adelante.*/
.m-cta-box__cta {
 position: absolute;
 right: 0;
 bottom: 0;
 left: 0;
 border-top-left-radius: 0;
 border-top-right-radius: 0;
 text-align: center;
}

.m-cta-box__content > * {
 margin-bottom: 0.5em;
}

Recomendaciones de estilo para moléculas

Evita definir tamaños y márgenes directamente en las moléculas. Como las moléculas siempre se van a utilizar dentro de organismos (estructuras más grandes), es preferible que los tamaños y márgenes se definan a nivel de organismo. De lo contrario, si estableces, por ejemplo, un max-width en la molécula, puedes limitar su flexibilidad. Si en algún momento necesitas usar esa molécula en un contexto más grande, tendrás que modificar los estilos.
