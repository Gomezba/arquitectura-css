Empezando a aplicar estilos propios

Elements

En esta capa empezamos a definir nuestros estilos. Aún no escribiremos ninguna clase, sino que definimos como se mostrarán los elementos. Lo más habitual será definir estilos de tipografía y enlaces.

Estilos elements SCSS

Ya hemos mencionado que las arquitecturas fueron creadas para aprovechar las ventajas de los preprocesadores como SASS o LESS, a continuación veremos como quedaria nuestra carpeta elements y nuestro main.css o index.css

_body.scss 

 @use "../settings/variables";
 
body {
  color: variables.$text-color;
  font-family: variables.$font-family;
  font-size: variables.$fs-body;
  line-height: 1.8;
}

_headings.scss 

 @use "../settings/variables";
 
h1,
h2,
h3 {
  font-weight: 300;
  line-height: 1.13;
}
 
h1 {
  font-size: variables.$fs-title-l;
}
 
h2 {
  font-size: variables.$fs-title-m;
}
 
h3 {
  font-size: variables.$fs-title-s;
}

_img.scss 

 img {
  display: block;
  max-width: 100%;
}

_links.scss 

 @use "../settings/variables";
 
a {
  color: variables.$primary-color;
  text-decoration: underline;
 
  &:hover,
  &:focus {
    color: variables.$primary-dark-color;
  }
}

index.scss 

 // Settings
@use "settings/colors";
@use "settings/layout";
@use "settings/typography";
@use "settings/z-index";
 
// Tools

// Generic

/*@use "generic/reset";*/

// Elements
@use "elements/body";
@use "elements/headings";
@use "elements/img";
@use "elements/links";

Directica @use en SASS

Cuando trabajamos con variables y mixins, es recomendable usar @use para que las variables solo sean accesibles en ese módulo de Sass, y no de forma global. Así siempre vemos claramente de donde salen las variables que estamos usando, y nos queda el código más modular y fácil de leer y mantener. Sass esta desfavoreciendo el uso de @import y en los próximos años lo eliminará del lenguaje, por lo que es mejor acostumbrarnos a usar @use cuanto antes.

Principales Características de @use

  • Importación de Módulos con Espacio de Nombres:Al importar un archivo SASS con @use, las variables, funciones y mixins del archivo importado se acceden mediante un espacio de nombres, que es el nombre del archivo por defecto o uno personalizado que se puede especificar. 

     // _colors.scss
$primary-color: #3498db;
$secondary-color: #2ecc71;
 
// elements.scss
@use '../settings/colors'; /*Ruta donde se encuentran nuestras variables*/
 
/*Accedemos a la variable accediendo desde colors.$variable*/
p {
    color: colors.$primary-color; 
}

  • Evitar la Contaminación del Espacio de Nombres: A diferencia de @import, @use no inyecta automáticamente todo el contenido del archivo importado en el archivo que lo está utilizando. Esto previene conflictos de nombres y asegura que las variables y mixins solo se utilicen cuando se hace referencia explícita a su espacio de nombres.

  • Modificación de Variables con @use: Si necesitas personalizar las variables de un módulo antes de usarlas, puedes hacerlo utilizando la opción with. Esto te permite sobrescribir las variables de configuración en el módulo importado. 

     // _colors.scss
$primary-color: #3498db;
$secondary-color: #2ecc71;
 
// elements.scss
@use '..settings/colors' with (
    $primary-color: #e74c3c
);
 
p {
    color: colors.$primary-color; // Utiliza el valor sobrescrito #e74c3c
}

  • Espacios de Nombres Personalizados: Puedes cambiar el espacio de nombres predeterminado utilizando la palabra clave as. 

     @use 'colors' as c;

body {
   background-color: c.$primary-color;
}

    Esto es útil si el nombre predeterminado es demasiado largo o si quieres evitar conflictos con otros módulos.

  • Reutilización de Módulos: @use permite reutilizar módulos SASS en diferentes partes de tu proyecto, manteniendo el código limpio y organizado.

Estilos elements CSS nativo

Anteriormente se enseñó la aplicación de de nuestros archivos de la carpeta elements pero con SASS.

Nosotros trataremos de utlizar CSS nativo, sin preprocesadores ya que actualmente tenemos POSTCSS lo cual podremos utilizar css nativo con las ventajas que tiene un preprocesador.

Así quedarian nuestros archivos dentro de la carpeta elements:

Una pequeña desventaja que tenemos al trabajar de manera pura, es que al momento de utilizar variables es que perdemos un poco de legibilidad al no tener la directiva @user de forma que no podemos saber de que archivo viene la variable.

Para poder leer variables que están en un archivo por aparte, en VSCODE tenemos la extensión CSS Variable Autocomplete.

Diferencia de archivos _archivo.scss archivo.css

Si nunca hemos trabajado con SASS puede que en lo que llevamos de este artículo nos hayamos hecho la pregunta de ¿Por qué los archivos scss tienen un ( _ ) al incio del nombre?

Archivos Parciales en SASS/SCSS:

  • Guion Bajo (_) en el Nombre del Archivo: En SASS/SCSS, los archivos que comienzan con un guion bajo (_) se consideran archivos parciales, por ejemplo, _variables.scss o _mixins.scss. Este guion bajo es una señal para SASS/SCSS de que el archivo es parcial y no debe ser compilado en un archivo CSS independiente.

  • No se Compila Directamente: Cuando SASS/SCSS compila tus archivos, los parciales no generan archivos CSS por sí mismos. En cambio, deben ser importados en un archivo SCSS principal que se compilará en CSS. Por ejemplo, si tienes un archivo main.scss que importa varios archivos parciales, solo main.scss se compilará en CSS.

  • Uso de @import o @use: Para incluir el contenido de un archivo parcial en otro archivo SCSS, usas la directiva @import o @use. Por ejemplo:

     // Archivo main.scss
@use 'variables';  // Utiliza _variables.scss
@use 'mixins';     // Utiliza _mixins.scss

    Aquí no necesitas incluir el guion bajo ni la extensión .scss en el @import, ya que SASS/SCSS los reconoce automáticamente.

  • Ejemplo de uso: 

     // _variables.scss
$primary-color: #3498db;
$font-stack: 'Helvetica, sans-serif';
 
// _mixins.scss
@mixin center-content {
  display: flex;
  justify-content: center;
  align-items: center;
}
 
// _base.scss
body {
  font-family: $font-stack;
  color: $primary-color;
}
 
// main.scss
@use 'variables';
@use 'mixins';
@use 'base';

    En este caso, main.scss es el archivo principal que importa todos los parciales (_variables.scss, _mixins.scss, _base.scss). Al compilar main.scss, se generará un solo archivo CSS que contiene todo el código de los parciales.
