Base CSS estilos globales

Hasta este punto ya hemos seteado todas las variables y algun mixin que necesitemos, ahora empezaremos a meter un poco de estilo.

Estilos Generic

Ya sabemos que la capa Generic es la primera en la que empezamos a generar CSS real que se aplica a nuestro documento. En esta capa, se incluyen estilos generales que afectan a la mayoría de los elementos, como los resets o normalize de CSS, que eliminan los estilos predeterminados del navegador y establecen una base consistente para todos los navegadores. También es común definir propiedades globales como box-sizing.

En este punto suele haber divisiones ya que hay quienes suelen optar por un reset u otros por un normalize.css

¿Porque suele haber estas divisiones?

Esto surge por el hecho de que los navegadores suelen tener sus propios estilos por defecto para cada elemento del navegador, pero esto ya depende de igual manera del sistema operativo que puede involucrar estilos por defecto dependiendo el navegador y así mismo con mac.

Aquí es donde entra en juego la división. Esto ya depende del enfoque que mejor se adapte a la forma de desarrollar un proyecto.

Reset

Un reset se encarga de eliminar por totalemente los estilos por defecto que asignan los navegadores, como es el padding y margin y las demás propiedades de cada etiqueta.

Es bueno dejar algunas propiedades ya que por algo se les asignan ciertos estilos a algunas etiquetas.

No es obligatorio seguir las normas de siempre aplicar un reset así de gordo, ya que un error muy común y muy mala práctica es quitarle los paddings y margins a todos los elementos. Esto es muy mala práctica ya que depués nos obliga a volver asignar los mismos estilos que reseteamos a cada elemento.

Podemos aplicar resets pequeños para etiquetas en concreto que verdaderamente necesitemos resetearlas, como la etiqueta body que se le suele resetar el margin a 0 para eliminar esos margenes que vienen por defecto, pero solo se lo aplicamos al body y no a todos los elementos.

Este es un reset demasiado gordo que elimina casi todos los estilos por defecto de los navegadores, esto suele ser muy mala práctica ya que después nos obliga a poner de nuevo los estilos que reseteamos como paddings y margins y ademas de eso se pierde la visualización semántica que deben tener las etiquetas, por ejemplo si queremos una lista vamos a querer que el usuario sepa que es una lista, un reset así de gordo nos obliga hacer un poco de doble trabajo.

Normalize.css

Un normalize de lo que se encarga es que nuestras páginas se visualizen de la misma forma en todos los navegadores, no aplica resets masivos solo los necesarios para que en todos los navgeadores se miren igual nuestros sitios.

Es mejor usar un normalize ya que partimos de una base.

Descargar hoja de estilos de normalize -> normalize.css

Box-sizing

En nuestra hoja de estilos de box-sizign quedaría de la siguiente manera: 

 *,
*::before,
*::after {
  box-sizing: border-box;
}
