Cómo utilizar variables, mixins y funciones

Empezando a declarar variables

En las capas de Settings y Tools dentro de nuestra arquitectura ITCSS, configuramos todas las variables y herramientas esenciales que usaremos en el proyecto. Esto incluye definir los colores, tipografías, mixins y funciones que nos ayudarán a mantener un código limpio y organizado.

Si no somos imaginativos con los nombres de colores, podemos usar herramientas como Name that color:

Name that color

Diferenciando variables privadas y públicas:

  • Variables privadas: Se usan para definir valores fundamentales, pero no se aplican directamente en el CSS. Estas variables suelen llevar un guion bajo al principio del nombre, indicando que son para uso interno.

  • Variables públicas: Estas son las que aplicamos directamente en los estilos CSS. Se definen utilizando las variables privadas, lo que nos permite mantener un código más claro y flexible.

Para estos ejemplos utilizaremos CSS nativo 

        :root{
/*Variables privadas*/    
--_eucalyptus: #289f5f;
--_green-pea: #19623b;
--_baltic-sea: #282729;
--_cod-gray: #181818;
--_dove-gray: #666;
--_boulder: #777;
--_white: #fff;
--_alabaster: #fafafa;
--_gallery: #ececec;

/*Estas variables privadas las usamos para definir las variables públicas:*/


--primary-color: var(--_eucalyptus);
--primary-dark-color: var(--_green-pea);

--background-dark-color: var(--_baltic-sea);
--background-darker-color: var(--_cod-gray);
--background-light-color: var(--_alabaster);
--background-lighter-color: var(--_white);

--text-color: var(--_cod-gray);
--text-muted-color: var(--_dove-gray);
--text-inverted-color: var(--_white);

--border-color: var(--_dove-gray);
--border-inverted-color: var(--_gallery);
}

Explicación

Definimos los colores base como variables privadas, tales como

--_eucalyptus o --_cod-gray.

A partir de estas, creamos variables públicas como

--primary-color: var(--_eucalyptus);

que se aplican directamente en los estilos.

Beneficios de esta práctica de utilizar variables

  • Reutilización y simplificación: Podemos identificar y eliminar colores similares que cumplan la misma función, haciendo el código más eficiente.

  • Facilidad de cambio: Si el equipo de diseño decide cambiar un color, al tener variables independientes para diferentes aplicaciones (como --background-darker-color y --text-color), podemos actualizar los valores fácilmente sin que el cambio afecte otros elementos del diseño.

Facilidad de cambio práctico

Supongamos que tienes las siguientes variables definidas:

 :root{
/*Variables privadas (colores base):*/    
--_eucalyptus: #289f5f;
--_cod-gray: #181818;
--_dove-gray: #666;
--_white: #fff;

/*Variables públicas (usadas en el CSS):*/
--primary-color: var(--_eucalyptus);
--background-darker-color: var(--_cod-gray);
--text-color: var(--_cod-gray);
--text-muted-color: var(--_dove-gray);
--text-inverted-color: var(--_white);
}

En este caso, tanto

--background-darker-color como --text-color están usando el valor --_cod-gray.

Supongamos que el equipo de diseño decide que el fondo oscuro (--background-darker-color) debe ser de un tono más claro, pero el color del texto (--text-color) debe permanecer igual.

Dado que --background-darker-color y --text-color son variables independientes, puedes cambiar solo --background-darker-color sin afectar el color del texto. 

 /* Nuevo valor para la variable privada*/
--_charcoal-gray: #333; // Un tono más claro

/* Actualización de la variable pública para el fondo*/
--background-darker-color: var(--_charcoal-gray);

/* Manteniendo el color del texto */
--text-color: var(--_cod-gray);

Ahora, tu fondo tiene un nuevo color (#333), pero el texto sigue usando el color anterior (#181818). Si hubieras usado la misma variable para ambos (fondo y texto), tendrías que modificar más lugares y quizás no podrías mantener el texto con el color original sin hacer ajustes adicionales.

Conclusión: Este enfoque te permite cambiar una parte específica del diseño (como el fondo) sin afectar otras (como el texto), lo que facilita realizar ajustes precisos según las indicaciones del equipo de diseño.

Los mixins y funciones de Sass o Postcss es importante que se definan cuando vemos que sea necesario, y no definir demasiados de entrada, ya que estaríamos creando abstracciones prematuras.

Aquí puedes aprovechar plugins como postcss-mixins para definir mixins o postcss-functions para funciones personalizadas. 

 /* tools/_mixins.css */
@define-mixin justify {
 display: flex;
 justify-content: center;
 align-items: center;
}
/*Cuando ocupemos el mixin seria de la siguiente manera*/
.container {
   @mixin justify;
}

Nuestro main.css o index.css quedaria de la siguiente manera hasta este momento.

 // Settings
@import "settings/colors.css";
@import "settings/layout.css";
@import "settings/typography.css";
@import "settings/z-index.css";
 
// Tools
@import "tools/justify.css";
