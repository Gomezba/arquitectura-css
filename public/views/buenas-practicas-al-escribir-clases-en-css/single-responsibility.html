Single responsibility principle en CSS

Entendiendo que es Single Reponsibility Principle en CSS

En CSS, al igual que en cualquier otro lenguaje, es útil seguir principios como el de Responsabilidad Única (Single Responsibility Principle), que forma parte de la S de SOLID. La dificultad radica en definir cuál es la responsabilidad específica de cada clase.

Un ejemplo común en cualquier proyecto web es la necesidad de convertir una lista (ul) para eliminar las viñetas y los márgenes o rellenos predeterminados. Queremos mantener la estructura semántica de la lista, pero cambiar su apariencia para que no se vean esos estilos predeterminados.

Aquí es donde entra en juego el concepto de ui-list. En desarrollo web, "ui-list" se refiere a una clase genérica utilizada para estilizar listas (ul) de manera consistente en toda la interfaz de usuario. Una clase genérica es aquella que se puede aplicar a múltiples elementos en diferentes partes de un sitio web, sin estar vinculada a un contexto o componente específico. Su responsabilidad principal es eliminar las viñetas y los márgenes, dejando la lista preparada para su personalización en diferentes contextos.

Es crucial comunicar claramente esta responsabilidad a través del nombre de la clase. Dado que estamos trabajando con un ul, un nombre comúnmente utilizado es ui-list. Sin embargo, es importante no mezclar propiedades que controlen el diseño (como el layout) con aquellas que afectan la apariencia (theming), como los colores de fondo o el texto. Mezclar estas propiedades en la misma clase haría que perdiera su responsabilidad única.

Clase correcta ✓

 .ui-list {
     list-style-type: none;
     padding-left: 0;
     margin-top: 0;
     margin-bottom: 0;
}

La clase ui-list esta teniendo su responsabilidad única que es resetear los estilos predeterminados de una lista.

Clase incorrecta ✗

 .ui-list {
     list-style-type: none;
     padding-left: 0;
     margin-top: 0;
     margin-bottom: 0;
     
     position: sticky;
     color: white;
     background-color: lightcoral;
}

La clase ui-list esta perdiendo su responsabilidad única que es resetear los estilos predeterminados de una lista, al agregarle propiedades de posicionamiento y propiedades de theming.

Contextos de responsabilidad única

La responsabilidad única en CSS depende del contexto.

En el caso de un ul, como hemos mencionado, es habitual reutilizarlo en diferentes listas, por lo que evitamos mezclar propiedades de theming para mantener su responsabilidad única.

Sin embargo, en clases para elementos más específicos, como un header, podemos incluir propiedades de posicionamiento como sticky, tamaño (width), y theming en una misma clase. En este caso, la responsabilidad de la clase es estilizar el encabezado principal de nuestra aplicación, y eso incluye tanto su layout como su apariencia.

 .header{
   width: 100%;
   position: sticky;
   top: 0;
   left: 0;
   background-color: lightblue;
   color: #fff;
}

En el contexto de un .header si es válido el mezclar propieades de (layout) y (theming) ya que estamos estilando el header principal y sus estilos son únicos.
